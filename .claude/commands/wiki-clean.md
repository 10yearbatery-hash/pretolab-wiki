팀 wiki 점검 및 정리 (기덕 전용, 주 1회)

다음 순서로 실행:

## 1. 파일 목록 수집

`.omc/wiki/` 의 실제 파일 목록을 가져온다 (index.md, log.md 제외).

## 2. index.md 파싱

`.omc/wiki/index.md` 를 읽어 등재된 파일명 목록을 추출한다.

## 3. 점검 항목 4가지

- **고아 파일**: 실제 존재하지만 index.md에 등재되지 않은 파일
- **깨진 링크**: index.md에 등재됐지만 실제 파일이 없는 항목
- **중복 등재**: 같은 파일이 index.md에 2회 이상 등재된 경우
- **중복 내용**: 제목 또는 내용이 겹치는 페이지 (파일을 읽어 비교)

## 4. 발견 목록 보고

발견된 항목을 표 형식으로 보여주고, 각각 처리 방법을 제안한다:
- 합치기: 두 페이지 내용을 하나로 통합 후 나머지 삭제
- 삭제: 오래됐거나 불필요한 파일 제거
- index 추가: 고아 파일을 index.md에 등재
- 유지: 내용이 달라서 둘 다 필요한 경우

## 5. 사용자 확인 후 처리

사용자가 승인하면 처리를 실행한다:
- 파일 삭제 시: `.omc/wiki/{파일}` 제거 + index.md 항목 제거 + 페이지 카운트 갱신
- 파일 병합 시: 내용 통합 후 기존 파일 삭제 + index.md 갱신
- 고아 파일 등재 시: index.md 해당 category 섹션에 항목 추가 + 카운트 갱신

## 6. 완료 후 git 동기화

```bash
git add .omc/wiki/
git commit -m "wiki: 주간 정리"
git push
```

완료 후: "✅ wiki 정리 완료. 팀원들이 /wiki-pull 하면 반영됩니다."
