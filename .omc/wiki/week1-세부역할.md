# 첫주차 세부 역할 분담 (6/2~6/8)

> 데모데이까지 가장 중요한 주. MVP 플로우 확정 직후 첫 실행 주차.
> 건준이 시험 전 일부 빌드 리서치 기여 가능 → 기덕 PRD 집중.

---

## 기덕 — PRD 작성 + 아키텍처 설계

| 할 일 | 비고 |
|-------|------|
| 화면별 상세 스펙 작성 | QR 화면, 카메라 화면, 갤러리 화면, 브라우저 뷰어 |
| 시그널링 플로우 다이어그램 | QR 생성 → 연결 → 스트리밍 → 종료 |
| 에러 케이스 정의 | 연결 실패, 카메라 권한 거부, 재연결 등 |
| 건준 리서치 결과 받아 기술 스택 확정 | 시그널링 서버, WebRTC 라이브러리 |
| Apple Developer 계정 확인/등록 | 없으면 바로 결제 ($99, 심사 1-3일) |
| Bundle ID + GitHub repo 결정 | 영환과 함께 |
| 6/8 미팅 전 PRD 초안 완성 | |

**PRD 먼저 쓰는 이유:** 시험기간(6/8~6/21) 혼자 빌드할 때 판단 비용 제로화. 팀 없어도 PRD 보고 결정 가능.

---

## 건준 — 빌드 리서치 + 베이스 세팅

| 할 일 | 비고 |
|-------|------|
| WebRTC iOS 라이브러리 옵션 조사 | LiveKit vs 네이티브 WebRTC.framework |
| 시그널링 서버 옵션 비교 | Firebase vs Supabase vs Ably |
| iOS Safari WebRTC 브라우저 수신 호환성 확인 | 찍히는 사람 쪽 |
| AdMob 리워드 광고 SDK 세팅 방법 조사 | 광고 전환 수익 모델 |
| Xcode 프로젝트 기본 베이스 생성 | Bundle ID, 폴더 구조 |
| WTP 인터뷰 대상자 2-3명 연락 + 진행 | 6/8 미팅 전까지 |
